# 朝日ナンバーズ練習ツール 仕様書 v0.3（実装済み仕様）

公開URL: https://urahim.github.io/numbers-practice/

## 背景・コンセプト

朝日新聞デジタルのゲーム「ナンバーズ」は1日5回しか遊べず、ウォームアップや反復練習ができない。
本ツールは本番の再現ではなく、「和を瞬時に見つける能力」（判断速度・視線移動・数字の組み合わせ認識）を鍛えるための練習アプリ。

## 画面構成

1. **設定画面**：盤面サイズ・お題モードを選んでスタート
2. **ゲーム画面**：60秒間のプレイ
3. **結果画面**：正解数・正答率・平均回答時間・消したブロック数を表示

## 設定項目

### 盤面サイズ
- 5×5（推奨・デフォルト）
- 6×6
- （3×3・4×4は廃止済み）

### お題モード
- **通常**：毎回「数字（6〜24）／奇数／偶数／素数」からランダムに1つを選ぶ
- **16だけ／20だけ／21だけ／23だけ**：固定の数字を目指す
- **奇数／偶数／素数**：選んだマスの合計がその性質を満たせば正解（特定の数字である必要はない）

## 盤面・お題の生成ロジック

- 盤面はゲーム開始時に1回だけランダム生成（各マス1〜9）
- お題は、現在の盤面の中から実際に作れる組み合わせ（2〜4マス）だけを出す（解けない問題は出さない）
- 固定数字モードは、盤面生成時にその数字を作れるか探索し、作れない盤面なら作り直す

## 操作（v0.3：ドラッグ選択）

- 指を触れたまま隣接マスをなぞって選択する（タップでの自由選択・決定ボタンは廃止）
- 1つ前のマスへ指を戻すと、そこまで選択が縮む（なぞり直し）
- 選択したマスの中心を結ぶ線を表示
- 選択中の合計を「7 + 8 + 6 = 21」の形式で表示
- **指を離した瞬間に自動判定**（選択2マス未満では何も起きない）

## 正解時の挙動

- スコア+1
- 正解した**そのマスだけ**盤面から消える（他のマスはそのまま）
- 消えたマスの列は、上から新しい数字が補充される（重力落下方式）
- 次のお題は、補充後の盤面から解けるものをランダムに選び直す
- アニメーション：正解マスが縮んで消える（約0.2秒）→ 影響を受けた列だけ上からフェードインして落ちてくる（アニメーション中は入力ロック）

## 不正解時の挙動

- 選択だけリセットされる（盤面は変化しない）。指を離すたびに仕切り直し
- 統計用に「判定した回数」としては裏でカウントされる（正答率の分母になる）

## 制限時間・終了後の統計

- 60秒固定
- 終了後に表示：
  - 正解数
  - 正答率（決定を押した回数のうち正解した割合）
  - 平均回答時間（正解ごとの反応時間の平均）
  - 消したブロック数（正解時に消えたマスの合計数）
- ボタン：「同じ設定でもう一度」「ホームに戻る」

## 見た目（数字ごとの色・形）

本番のナンバーズを参考に、数字ごとに固定の色・形を割り当て（彩度は控えめ）。

| 数字 | 形 | 色 |
|---|---|---|
| 1 | 丸 | 赤茶 |
| 2 | 丸 | 水色 |
| 3 | 三角 | 緑 |
| 4 | 四角 | 黄土色 |
| 5 | 星（丸みあり） | 黄色（文字は濃色） |
| 6 | 六角形 | ピンク |
| 7 | 星（丸みあり） | 紫 |
| 8 | 六角形 | 黄緑（文字は濃色） |
| 9 | 星（9頂点・丸みあり） | 紺色 |

- 選択中のマス：暗くする＋白い光彩の縁取り（明るくするのではなく、暗くして区別）。形（clip-path）の外にも縁取りが出るよう、形の描画は疑似要素(::before)に分離している
- なぞって次に選べないマス（非隣接）：半透明（グレーアウト）

## 技術構成

- Web技術のみ（HTML / CSS / JavaScript、フレームワーク不要）
- PWA対応（`manifest.json` + `service-worker.js`）。オフラインキャッシュがあるため、更新後は端末側でキャッシュ削除が必要な場合あり
- ホスティング：GitHub Pages（`urahim/numbers-practice`、`main`ブランチから自動デプロイ）
